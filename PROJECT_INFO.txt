================================================================================
  PROJECT OVERVIEW
================================================================================

Target Frenzy 3D is a complete, playable 3D battle royale arena shooter game
built entirely from scratch in Python using OpenGL for real-time 3D rendering.
The game demonstrates proficiency in real-time graphics programming, game
engine architecture, AI systems, physics simulation, and interactive design
— all within a single self-contained Python file.

The player is placed in a massive arena and must survive waves of intelligent
enemies that shoot back, while the battle royale zone shrinks over time. The
game features a multi-weapon system, power-ups with unique 3D visualizations,
headshot mechanics, combo scoring, kill streaks, and a fully featured HUD with
minimap.

================================================================================
  TECHNICAL SKILLS DEMONSTRATED
================================================================================

1. REAL-TIME 3D GRAPHICS (OpenGL)
   - Fixed-function OpenGL pipeline (GL_LIGHTING, GL_FOG, GL_BLEND)
   - Custom 3D models built from primitives (glutSolidCube, gluSphere,
     glutSolidOctahedron, glutSolidTorus, gluNewQuadric)
   - Perspective projection with gluPerspective and gluLookAt
   - First-person overlay rendering (clearing depth buffer for FP gun)
   - 2D HUD overlay using orthographic projection
   - Particle effects system with physics
   - Translucent rendering with alpha blending
   - Per-vertex coloring with GL_COLOR_MATERIAL
   - Ambient/diffuse/specular lighting setup
   - Atmospheric fog for depth perception
   - Smooth shading (GL_SMOOTH)

2. GAME ENGINE ARCHITECTURE
   - Complete game loop: input → update → render
   - Delta-time based updates for frame-rate independence
   - 60 FPS frame limiter for consistent performance
   - State machine for game flow (Name Entry → Guidelines → Playing →
     Paused → Game Over)
   - Modular function design (~60+ functions organized by system)
   - Global state management with careful scoping

3. GAME PHYSICS
   - Momentum-based player movement with friction
   - Delta-time scaling for consistent physics across frame rates
   - AABB collision detection for crate obstacles
   - Spherical collision for pillar obstacles and projectiles
   - Axis-sliding for wall collision response
   - Push-out resolution when stuck inside obstacles
   - Diagonal movement normalization

4. ARTIFICIAL INTELLIGENCE (4 ENEMY TYPES)
   - Grunt: Direct chase behavior with moderate shooting
   - Tank: Slow advance with heavy damage and high HP
   - Scout: Zigzag dodging with fast movement and direction changes
   - Sniper: Range maintenance (retreats if too close, advances if too far)
   - All enemies: Shoot at player with type-specific accuracy/spread
   - Dynamic difficulty scaling (+12% HP per level)
   - Fire rate randomization for natural feel

5. WEAPON SYSTEM
   - 3 distinct weapons: Pistol, Assault Rifle, Shotgun
   - Each weapon has unique: damage, fire rate, spread, ammo capacity,
     reload time, projectile count (pellets), visual color
   - Real-time weapon switching with 1/2/3 keys
   - Reload mechanic with timer and auto-reload on empty
   - Ammo management with kill-based replenishment
   - Unique first-person weapon models per weapon type:
     * Pistol: Compact slide/frame/grip model
     * Assault Rifle: Full receiver/barrel/magazine/sights
     * Shotgun: Long barrel, tube magazine, pump grip, stock
   - Weapon-specific projectile rendering:
     * Pistol: Elongated bullet shape
     * Assault Rifle: Tracer with glow effect
     * Shotgun: Small sphere pellets

6. COMBAT MECHANICS
   - Headshot detection system based on projectile height vs enemy zones
   - Probability-based headshot confirmation (25% in head zone)
   - Hit marker system: white X for body, red X for headshots
   - Floating damage numbers with upward drift animation
   - Kill confirmation floating text (+points)
   - Muzzle flash overlay effect
   - Enemy-to-player melee damage on close range
   - Shield absorption mechanic (damage → shield first → HP)

7. SCORING & PROGRESSION
   - Combo system: chain kills within 3s for up to 5× multiplier
   - Kill streak tracking with milestone bonuses:
     * 5 kills: "RAMPAGE!" (+50)
     * 10 kills: "UNSTOPPABLE!" (+100)
     * 15 kills: "GODLIKE!" (+200)
   - Headshot score bonus (1.5× points)
   - Level/wave system: 10+ kills per level to advance
   - Increasing enemy count and variety per level
   - Post-game statistics: score, level, kills, streaks, combo,
     time survived, accuracy percentage

8. POWER-UP SYSTEM WITH 3D VISUALIZATION
   - 4 power-up types with distinct 3D shapes:
     * Health: Green 3D cross/plus shape
     * Speed: Blue spinning octahedron with speed lines
     * Damage: Red spiky star (sphere with spike octahedra)
     * Shield: Yellow translucent dome with rotating torus ring
   - Pulsing glow rings underneath each power-up
   - Bobbing and spinning animations
   - Timed spawning system
   - Collection with particle effects

9. BATTLE ROYALE ZONE
   - Shrinking zone circle with configurable intervals
   - Zone damage to players and enemies outside boundary
   - Translucent zone wall visualization
   - Dynamic zone center shifting
   - HUD warning notifications
   - Minimap visualization of zone and target zone

10. HUD & UI SYSTEM
    - Health bar with color-coded thresholds (green → yellow → red)
    - Shield bar (appears when active)
    - Stamina bar for sprint management
    - Score display with combo multiplier
    - Level and kill progress tracker
    - Player name display
    - Active power-up timers
    - Kill streak labels
    - Weapon indicator: name, ammo bar, slot selector [1][2][3]
    - Reload progress indicator
    - First-person crosshair with dynamic hit markers
    - Minimap showing: player, enemies, power-ups, obstacles, zone
    - Damage flash overlay (red screen flash)
    - Level-up splash notification
    - Floating notification system (right side)
    - Zone shrinking warning

11. CAMERA SYSTEM
    - First-person view: eye at head height, looking in facing direction
    - Third-person view: camera behind/above player, same look direction
    - Smooth camera following
    - Configurable zoom (distance/height) via arrow keys
    - Gun model overlay in first-person (depth-cleared)

12. INPUT HANDLING
    - Full keyboard controls (WASD + QE movement)
    - Smooth rotation with momentum interpolation
    - Sprint toggle with Shift held
    - Key-up detection for smooth movement stops
    - State-aware input (different behavior per game state)
    - Name entry text input with backspace support
    - Special key handling (arrows) for camera zoom

================================================================================
  KEY DESIGN DECISIONS
================================================================================

1. Single-file architecture: Keeps the entire game engine self-contained for
   portability and easy distribution. No external assets needed.

2. OpenGL fixed-function pipeline: Demonstrates understanding of core graphics
   concepts (transformations, lighting, blending) without shader abstraction.

3. Procedural 3D models: All models (player, enemies, weapons, power-ups) are
   built from OpenGL primitives, showing geometric composition skills.

4. Delta-time physics: All movement and animations use delta-time scaling for
   consistent behavior regardless of frame rate.

5. FPS limiter: Prevents excessive CPU usage while maintaining smooth gameplay
   at 60 FPS.

6. State machine for game flow: Clean separation of game states (menu, playing,
   paused, game over) with state-specific input handling.
